## üë• Managing Team Members

### Adding a New Team Member

1. Go to the **Admin Panel** ‚Üí **Team Members** tab
2. Click **"+ Add Team Member"**
3. Fill in the fields:
   - **Name:** Full name (e.g., "Dr. Sarah Chen")
   - **Role:** Job title (e.g., "Principal Investigator")
   - **Bio:** Short description (2-3 sentences)
   - **Expertise:** Comma-separated list (e.g., "Data Ethics, Privacy, AI Policy")
   - **Image URL:** Link to profile photo
4. Click **"Save"**

### Getting Image URLs

**Option 1 - Upload to Unsplash-style service:**
- Upload to [Imgur](https://imgur.com) (free)
- Right-click image ‚Üí "Copy image address"
- Paste into Image URL field

**Option 2 - Use existing photos:**
- Right-click on any web photo
- Select "Copy image address"
- Paste the URL

### Editing a Team Member

1. Find the team member card
2. Click the **pencil icon** (‚úèÔ∏è)
3. Update the fields
4. Click **"Save"**

### Deleting a Team Member

1. Find the team member card
2. Click the **trash icon** (üóëÔ∏è)
3. Confirm deletion

## üìù Managing Blog Posts

### Creating a New Blog Post

1. Go to **Admin Panel** ‚Üí **Blog Posts** tab
2. Click **"+ Add Blog Post"**
3. Fill in:
   - **Title:** Post headline
   - **Excerpt:** Short summary (1-2 sentences)
   - **Author:** Name of the author
   - **Author Image URL:** Photo of author
   - **Date:** Publication date (format: YYYY-MM-DD)
   - **Read Time:** Estimated time (e.g., "8 min read")
4. Click **"Save"**

### Tips for Good Blog Posts

- **Title:** Keep it under 60 characters for best display
- **Excerpt:** Make it compelling - this appears on the blog page
- **Date:** Posts are sorted by date (newest first)
- **Read Time:** Aim for 5-10 minute reads

## üî¨ Managing Research Projects

### Adding a New Project

1. Go to **Admin Panel** ‚Üí **Research** tab
2. Click **"+ Add Project"**
3. Fill in:
   - **Title:** Project name
   - **Description:** What the project is about (2-3 sentences)
   - **Status:** Choose from dropdown (Active, Planning, or Completed)
   - **Tags:** Comma-separated keywords (e.g., "Privacy, Ethics, Methodology")
   - **Image URL:** Relevant project image
4. Click **"Save"**

### Project Status Guide

- **Active:** Currently ongoing research
- **Planning:** In development, not yet started
- **Completed:** Finished projects with results

## üí° Pro Tips

### Writing Good Descriptions

‚úÖ **Good:**
"Developing comprehensive metrics to assess individual and collective digital well-being across various platforms and contexts."

‚ùå **Avoid:**
"This project is about metrics."

Keep descriptions:
- Clear and specific
- 2-3 sentences
- Jargon-free (or explain technical terms)
- Action-oriented

### Organizing Content

**Team Members:**
- Order matters! List senior members first
- Keep expertise tags consistent across team

**Blog Posts:**
- Date posts accurately - they auto-sort by date
- Use consistent author names

**Research Projects:**
- Use consistent tag names across projects
- Review and update project status regularly

**Partners:**
- Group similar partner types together
- Keep descriptions concise but informative

### Image Best Practices

**Size:**
- Team photos: Square format works best (e.g., 400x400px)
- Blog author photos: Small squares (e.g., 100x100px)
- Project images: Landscape format (e.g., 800x500px)

**Quality:**
- Use high-resolution images
- Ensure images are clear and professional
- Compress images for faster loading

**Hosting:**
- Use reliable image hosts (Imgur, Cloudinary, etc.)
- Don't use temporary image links
- Test image URLs before saving

## ‚ö†Ô∏è Common Issues

### "Save" Button Not Working

**Solutions:**
1. Check that all required fields are filled
2. Ensure image URLs are valid
3. Check browser console for error messages
4. Try refreshing the admin panel

### Content Not Showing on Site

**Solutions:**
1. Click "Save" and wait for confirmation
2. Refresh the main website
3. Clear browser cache (Ctrl+Shift+R / Cmd+Shift+R)

### Lost Changes

**Prevention:**
- Always click "Save" after editing
- Don't close the tab while saving
- Consider copying long text before submitting

## üÜò Need Help?

1. **Can't access admin panel?**
   - Verify the URL includes `/#admin`
   - Check if JavaScript is enabled in browser
   - Try a different browser

2. **Database won't initialize?**
   - Refresh the page
   - Check browser console for errors
   - Contact technical support

3. **Images won't load?**
   - Verify the URL works in a new browser tab
   - Use HTTPS URLs (not HTTP)
   - Try a different image host

4. **Content looks wrong on mobile?**
   - The site is responsive - this is expected
   - Test on actual mobile devices
   - Content adapts automatically to screen size
